MVC

Model Vue Controller

Présentation

MVC (acronyme de Model - Vue - Controller) n’est pas à proprement parlé un design pattern et il est plus généralement classé dans la catégorie des pattern d’architecture (architectural patterns).

Les pattern d’architecture sont plus généraux et peuvent être développés en utilisant en interne un ou plusieurs design patterns.

Par exemple, le design pattern MVC peut utiliser :

  • Observer : Pour simplifier l’interaction des objets entre eux
  • Composite : Pour gérer les éléments à afficher et leur imbrication
  • Strategy : Pour gérer les interactions entre le Contrôleur et la Vue

Avantages

Les avantages liés à l’utilisation de l’architecture MVC sont nombreux mais c’est surtout le couplage faible entre le modèle, la vue et le contrôleur qui est particulièrement intéressant.
